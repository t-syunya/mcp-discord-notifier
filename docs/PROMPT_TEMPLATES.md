# MCP Discord Notifier - プロンプトテンプレート集

このドキュメントは、Claude Code、Cursor、Codex等のAI開発アシスタントでMCP Discord Notifierを効果的に使用するためのプロンプトテンプレートを提供します。

---

## 基本的な使い方

AIアシスタントに以下のようなプロンプトを送ることで、自動的にDiscord通知機能が活用されます。

---

## 📝 作業記録・進捗報告

### テンプレート1: 作業開始時

```
今から[タスク名]を開始します。
作業の進捗をDiscordに記録しながら進めてください。
重要なマイルストーンに到達したら都度報告してください。
```

**例:**
```
今からユーザー認証機能の実装を開始します。
作業の進捗をDiscordに記録しながら進めてください。
重要なマイルストーンに到達したら都度報告してください。
```

---

### テンプレート2: 複数タスクの管理

```
以下のタスクを順番に実行してください：

3. [タスク3]

各タスクの開始時と完了時にDiscordに記録してください。
```

**例:**

```
2. APIエンドポイントの追加

```

```

調査の過程と発見した内容をDiscordに記録してください。

```

**例:**

## ✅ ユーザー確認が必要な作業

### テンプレート4: 破壊的な変更前の確認

```
[実行する作業]を実行しますが、これは[影響範囲]に影響します。
実行前にDiscordで確認を求めてください。
承認されたら実行してください。
```

**例:**
```
データベースのマイグレーションを実行しますが、これは本番環境のデータ構造に影響します。
実行前にDiscordで確認を求めてください。
承認されたら実行してください。
```

### テンプレート5: 複数の選択肢から選ぶ

```

- 選択肢A: [説明]
- 選択肢B: [説明]
- 選択肢C: [説明]

Discordで私に選択を求めてください。
```

**例:**
```
パフォーマンス改善について、以下の選択肢があります：
- 選択肢A: キャッシュを導入（実装時間: 2時間）
- 選択肢B: クエリの最適化（実装時間: 1時間）
- 選択肢C: インデックスの追加（実装時間: 30分）

Discordで私に選択を求めてください。
```

## 🔄 継続的な監視・定期実行

## 🎯 プロジェクト別のカスタムプロンプト

### Web開発プロジェクト用

```
Webアプリケーションの開発を進めます。

作業ルール：
- 新しい機能を追加する前に、Discordで確認を求める
- コンポーネントの実装完了時にDiscordに記録
- ビルドエラーが発生したら音声で通知
- テストが完了したら音声で通知
- デプロイ前に必ずDiscordで確認を求める

それでは、[具体的なタスク]を開始してください。
```

---

### データ処理・スクリプト実行用

```
データ処理タスクを実行します。

作業ルール：
- 処理開始時にDiscordに記録
- 処理の進捗を10%ごとにDiscordに報告
- エラーが発生したら音声で緊急通知
- 処理が完了したら音声で通知し、結果のサマリーをDiscordに記録

それでは、[データ処理タスク]を実行してください。
```

---

### インフラ・デプロイ作業用

```
インフラの変更作業を行います。

作業ルール：
- 各ステップの実行前に、作業内容と影響範囲をDiscordに記録し、確認を求める
- コマンドの実行結果をDiscordに記録
- エラーが発生したら即座に音声で通知
- すべての作業が完了したら音声で報告

それでは、[インフラ作業の内容]を開始してください。
```

---

### リファクタリング・コードレビュー用

```
コードベースのリファクタリングを行います。

作業ルール：
- 変更するファイル一覧をDiscordに記録
- 大きな変更を加える前に、変更内容をDiscordで確認を求める
- 各ファイルのリファクタリング完了時にDiscordに記録
- すべてのテストが通ったら音声で通知

それでは、[リファクタリング対象]のリファクタリングを開始してください。
```

## 💡 ベストプラクティス

### 1. **明確な指示を出す**
❌ 「よろしくお願いします」
✅ 「Discordに記録しながら進めてください」

### 2. **通知のタイミングを指定**
❌ 「通知して」
✅ 「完了したら音声で通知してください」

### 3. **確認が必要な場面を明示**
❌ 「実行してください」
✅ 「実行前にDiscordで確認を求めてください」

### 4. **通知レベルを使い分ける**
- **テキスト記録**: 通常の進捗、ログ
- **確認要求**: 重要な判断、破壊的変更
- **音声通知**: 完了通知、エラー・警告

### 5. **コンテキストを提供**
```
現在、本番環境のメンテナンス作業中です。
慎重に作業を進め、各ステップでDiscordに記録してください。
```

## 🚀 セッション開始時のおすすめプロンプト

作業開始時に以下のプロンプトを送ることで、AIがDiscord通知を適切に活用します：

```
今日の作業を開始します。

通知ルール：
- 重要な作業の開始・完了はDiscordに記録
- 確認が必要な場合はDiscordでリアクションを求める
- 長時間タスク（5分以上）が完了したら音声で通知
- エラーや警告は音声で通知

この通知ルールを守りながら、以下のタスクを進めてください：
[今日のタスク一覧]
```
