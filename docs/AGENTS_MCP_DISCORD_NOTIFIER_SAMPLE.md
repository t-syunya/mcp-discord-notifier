## Discord通知 & 音声合成 (mcp-discord-notifier)

### 概要

mcp-discord-notifierサーバーを通じて、Discordへのログ送信・リアクション待機・VOICEVOXによる音声通知を利用できます。作業の進捗報告や重要な通知をテキストと音声の両方で伝えられます。

### 設定情報

- **ログ出力先**: Discordテキストチャンネル（`LOG_CHANNEL_ID`）に自動でスレッドを作成
- **ログスレッド名**: `Conversation Log`（`LOG_THREAD_NAME`で変更可）
- **ボイスチャンネル**: `VOICE_CHANNEL_ID` で指定（音声通知時に利用）
- **音声合成エンジン**: VOICEVOX（`VOICEVOX_URL`、デフォルト: `http://localhost:50021`）
- **デフォルト話者**: 四国めたん（ノーマル）
- **音声ID (speaker_id)**: 1
- **通知優先度**: `normal` または `high`

### 使用方法

```jsonc
// mcp__mcp-discord-notifier__notify_voice ツールの呼び出し例
{
  "message": "ビルドが完了しました",
  "priority": "normal", // オプション（default: normal）
  "speaker_id": 1       // オプション（default: 1 = 四国めたん ノーマル）
}
```

### 音声出力が必須の場面

以下の場面では**必ず音声で通知**してください：

1. **タスクの開始時**
   - 例: 「〜の作業を開始します」
2. **タスクの完了時**
   - 例: 「〜が完了しました」
3. **エラー発生時**
   - 例: 「エラーが発生しました。〜を確認してください」
4. **長時間処理の開始前**
   - 例: 「これから〜の処理を実行します。少々お待ちください」

### 使用制限

- ボイスチャンネル接続とVOICEVOXエンジン起動が必要です。準備できていない場合、音声通知はエラーになります。
- テキストログと音声再生は常にセットで行われます（片方だけにはできません）。
- 作業の進捗、発見、困っていること、考えていることなど、自由に発言・記録してください。
